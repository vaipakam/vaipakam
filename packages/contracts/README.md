# @vaipakam/contracts

**Per-facet ABI JSONs + per-chain deployments + typed loaders — the single source of truth every monorepo app consumes.**

[![Workspaces typecheck](https://github.com/vaipakam/vaipakam/actions/workflows/ci.yml/badge.svg?branch=main)](https://github.com/vaipakam/vaipakam/actions/workflows/ci.yml)

## What is this

This package is the **bridge between `contracts/` (Solidity source) and every off-chain consumer** in the monorepo — `apps/defi`, `apps/www`, `apps/agent`, `apps/keeper`, `apps/indexer`. They all import ABIs + deployment addresses from here, not from individual hand-typed declarations. That keeps every Worker / frontend / bot ABI-aligned with whatever the contracts actually shipped.

What it carries:

- `src/abis/*.json` — per-facet ABI JSONs, regenerated by `contracts/script/exportFrontendAbis.sh` after any contract change.
- `src/abis/_source.json` — provenance stamp (contracts commit hash).
- `src/deployments.json` — consolidated per-chain Diamond + facet addresses, keyed by `chainId`.
- `src/deployments.ts` — typed loader (`getDeployment(chainId)`); the entry point every consumer uses.

**Non-goals:** this package doesn't carry Solidity source, doesn't run forge, doesn't deploy anything. It's a sync target only.

## How to run

No dev loop — this is a consumed-only package. Edit its files via the export scripts:

```bash
cd contracts
forge build && bash script/exportFrontendAbis.sh
bash script/exportFrontendDeployments.sh
```

Both scripts write into `packages/contracts/src/`.

## How to test

```bash
pnpm --filter @vaipakam/contracts exec tsc --noEmit
```

CI also exercises this package via every consumer's typecheck — if an ABI shape changes, downstream `apps/*` typechecks surface the mismatch immediately.
