## Thread — Intent-based swap-to-repay GA: Fusion resolver-pickup bridge (T-090 v1.1 GA, PR #<n>)

Closes #426. Completes the v1.1 surface that landed across #420 / #421 / #423 / #425 — the GA-completing piece the v1.1 launch deliberately deferred.

The agent worker's `POST /intent/fusion/post` endpoint now forwards the committed order to 1inch's Fusion resolver-pickup endpoint with the operator-held `INTENT_FUSION_API_KEY` secret server-side and passes the upstream response through to the dapp. With this in place, Fusion's solver network sees fresh commits and can fill them; without it (the v1.1 launch state) commits had to be cancelled after the deadline because there was no discovery path. The atomic surface stays available for borrowers who want predictable timing — the intent surface now offers solver-competition pricing in the minute-scale auction window.

Operator changes that this PR introduces:

- `apps/agent/wrangler.jsonc` declares a new `INTENT_FUSION_API_KEY` secret-store binding. Operator rotates the value in via `wrangler secret put INTENT_FUSION_API_KEY` after creating a Fusion API key on the 1inch developer portal.
- Per-chain support follows whichever chains 1inch's Fusion API itself supports; the request URL routes via `${FUSION_BASE_URL}/${chainId}/order/submit`.

If the secret is unset on a given deploy (e.g. the operator deploys the code change before rotating the secret), the endpoint degrades to the original queued-ack so the dapp's resolution path stays clean — exactly the v1.1 launch shape. Once the secret is rotated in, the real upstream `fetch` runs without any dapp redeploy.

The dapp's v1.1-alpha banner inside `SwapToRepayIntentPanel` is removed; the surface is now GA. The "Best-price intent" copy in the Advanced User Guide drops the "(v1.1 alpha)" qualifier and the alpha-status disclosure paragraph. The functional spec `SwapToRepayIntent.md` is updated to describe the agent bridge as the production-ready discovery path with the operator-pre-rotation fallback as an explicit edge case.
